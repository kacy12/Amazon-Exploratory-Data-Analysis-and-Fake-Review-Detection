Program Usage(it is important to run ETL_all before all other programs):

ETL_all.py [ratings/reviews]
ETL_all.py [review_meta] [reviews] [meta]
rating_price.py [single/all] [input] [output]
rating_score.py [single/all] [input] [output]
rating_stats.py [single/all] [input] [category](optional) [start_date] [end_date]
brand_effect_analysis.py [single/all] [input] [category](optional) [start_date] [end_date]
count_brand.py [single/all] ([category]) [input] [start_date] [end_date]
popularity_trends.py [input] [start_date] [end_date]
fake_review_detection.py review_meta [start_date] [end_date]
fake_review_graph.py [detail_input] [interval_input] [port]
correlation.py review_meta
featured_user_graph.py [port]
PercentageByID.py [review_meta] [ID]
User_Percentage.py [review_meta]
Top10ReviewItems.py [single/all] [input] [category](optional) [start_date] [end_date]



Commandline examples of how to run our program on cluster
(or locally if hadoop and spark are configured properly):

>spark-submit ETL_all.py ratings
>spark-submit ETL_all.py reviews
>spark-submit ETL_all.py review_meta reviews meta

At this point, all the data will be ready for the following executions.

>spark-submit rating_stats.py all review_meta 2003-01-01 2014-12-31
>spark-submit rating_stats.py single review_meta Baby 2003-01-01 2014-12-31

>spark-submit rating_price.py single review_meta rating_price
>spark-submit rating_price.py all review_meta rating_price

>spark-submit rating_score.py single review_meta rating_score_cate
>spark-submit rating_score.py all review_meta rating_price_all

>spark-submit brand_effect_analysis.py single review_meta Baby 2003-01-01 2014-12-31
>spark-submit brand_effect_analysis.py all review_meta 2003-01-01 2014-12-31

>spark-submit brand_count.py single Baby review_meta 2003-01-01 2014-12-31
>spark-submit brand_count.py all review_meta 2003-01-01 2014-12-31

>spark-submit popularity_trends.py review_meta 2003-01-01 2014-12-31

>spark-submit fake_review_detection.py review_meta 2004-01-01 2013-12-31

>python3 fake_review_graph.py fake_detail_2004-01-01_2013-12-31 fake_interval_2004-01-01_2013-12-31 8050

>python3 featured_user_graph 8051

>spark-submit correlation.py review_meta

>spark-submit PercentageByID.py review_meta A16CZRQL23NOIW

>spark-submit User_Percentage.py review_meta

>spark-submit Top10ReviewItems.py single review_meta Movies_and_TV 2009 2013

>spark-submit Top10ReviewItems.py all review_meta 2009 2013

In the dash environment, the front web can be executed as following and the data is all from the above executions.

>python3 featured_user_graph.py
>python3 plot_rating.py
>python3 plot_brand.py
>python3 plot_popularity.py
